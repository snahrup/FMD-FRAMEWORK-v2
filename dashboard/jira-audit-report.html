What Happened & Why

Root Cause: The FMD framework deployment from 2/17 needed to be completely rebuilt on 2/24. The INTEGRATION CONFIG workspace from the original deployment retained stale GUIDs in its SQL metadata database. When the dashboard integration phase began, these stale references caused cascading failures. The decision was made to delete all 5 INTEGRATION workspaces and start from scratch with a hardened deployment notebook.

Three Issues Uncovered

Fabric Notebook Cell Format — Fabric's REST API requires cell source to be an array of strings, not a concatenated string. The wrong format is silently accepted but produces a blank notebook. Cells 9 and 14 in NB_SETUP_FMD had this issue.
Stale GUID Conflict — The INTEGRATION CONFIG workspace from the 2/17 deployment was not deleted with the other workspaces. Its SQL database contained old workspace GUIDs that conflicted with the new deployment. The notebook failed at ~30 minutes trying to reconcile mismatched metadata.
Fabric CLI (fab) Subprocess Hang — The root cause of EVERY previous automated deployment failure. subprocess.run(["fab", ...]) hangs indefinitely at selector.select() when reading SQL database properties via get_item_id(). The database is created successfully, but the notebook locks up trying to read back the server FQDN. Workaround: use REST API to fetch properties directly.

Impact & Timeline Shift

All downstream dates shifted by approximately 1 week to account for the redeployment window. The critical path remains unchanged:

Source Registration (2/28) → Pipeline Testing (3/3) → Silver Layer (3/7) → Gold Layer (3/13) → E2E Testing (3/20) → Production Cutover (3/31)

Gateway connections (MT-15) survived as they are tenant-level. Source registrations (MT-16, MT-30-33) need re-doing but the dashboard wizard reduces this from hours to minutes per source.

Improvements for Future Deployments

Deployment Overlay — Dashboard now shows real-time deployment progress. Auto-discovers workspace GUIDs by name. Auto-applies config when notebook finishes. Zero manual GUID hunting required.
Cell Format Validation — NB_SETUP_FMD now has all cells in array-of-strings format (Fabric requirement). Previous string format was silently accepted but produced blank notebooks.
Admin Group Wiring — Service principal and FabricAdmins security group are pre-configured in workspace roles, domain contributors, and connection owners during setup.
Known Issue: fab CLI Hang — The Fabric CLI hangs on subprocess.run() when reading SQL database properties. Root cause of every automated failure. Workaround: pull properties via REST API. Permanent fix: replace all fab CLI calls with direct REST API equivalents in the notebook.
