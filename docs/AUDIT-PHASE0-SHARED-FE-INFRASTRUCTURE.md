## TOP 5 SYSTEMIC RISKS

1. **No ErrorBoundary (SHARED-FE-001)**: Single biggest systemic risk. Any uncaught error in any page = total app crash. Every page audit will inherit this risk.
2. **No 404 route (SHARED-FE-002)**: Navigation to undefined paths silently renders nothing. Combined with lazy loading failures, this creates invisible failure modes.
3. **No fetch cancellation (SHARED-FE-005)**: All 4 data hooks fire HTTP requests that are never cancelled on unmount. With polling enabled, fast navigation creates request pileup.
4. **Eager imports bloat bundle (SHARED-FE-004)**: ~40 synchronous page imports in App.tsx defeat code-splitting. Initial load downloads the entire application.
5. **Mock data with fake sources (SHARED-FE-003)**: If any page falls back to mockData.ts, users see fabricated Salesforce/SAP/Oracle data instead of real MES/ETQ/M3/OPTIVA data.
