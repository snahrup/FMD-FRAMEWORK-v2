# Gold Studio — Review Gate Template

Claude must return this structure after every packet.

## 1. Summary of implementation
- What was added
- What was updated
- What was intentionally left out

## 2. Files changed
- List files created
- List files modified
- List files removed, if any

## 3. Data / migration impact
- Migrations added or updated
- Schema assumptions
- Rollback concerns

## 4. API impact
- Endpoints added or touched
- Request/response schema changes
- Auth/role guard changes

## 5. UI impact
- Pages/components added or changed
- Shared components introduced
- Visual or interaction notes

## 6. Verification steps
- Commands to run
- Pages/routes to open
- Demo steps
- Expected visible behavior

## 7. Acceptance criteria checklist
- Criterion 1: pass/fail
- Criterion 2: pass/fail
- Criterion 3: pass/fail
- etc.

## 8. Known risks / TODOs
- anything incomplete
- anything stubbed
- anything that could surprise the next packet

## 9. Blocking questions
Only real blockers. No speculative redesign questions.
